When you enable SSO old OAuth tokens are not safelisted for use, so you will need to re-create them after enabling SSO with the following steps:

  • user logs in to the affected org on GitHub using SSO.
  • user goes to OAuth application settings,Â https://github.com/settings/applications, selects CircleCI token, revokes it
  • user goes to circleci.com, logs out if necessary, logs in again, which should request authorization, makes sure CircleCI is authorized to access the org
