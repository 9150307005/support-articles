If you find a build which doesn't use contexts even if you configure the context option in your workflow, you might be used v1.1 single job API to trigger builds.

If you see workflow config like the below, you hit this case.

workflows:
  version: 2
  build:
    jobs:
    - build: {}

Context is a part of workflow feature, and this is why the context isn't applied to the job.

So, you can use following APIs to trigger builds as workflow or pipeline, and it enables to use the context feature.

  • https://circleci.com/docs/api/v1/#trigger-a-new-build-by-project-preview
  • https://circleci.com/docs/api/v2/#operation/triggerPipeline
